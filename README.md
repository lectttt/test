## 如何使用 / How to Use

## 工作流程 / Workflow

1.  **探索 (Exploration)**: 我会首先查看你的代码库，阅读 README、代码文件和配置文件，以理解项目背景。
    *   I'll start by exploring your codebase, reading READMEs, code files, and configs to understand the context.
2.  **制定计划 (Planning)**: 我会提出一个详细的执行计划并请求你的审核。
    *   I'll propose a detailed execution plan and request your review.
3.  **执行任务 (Execution)**: 在计划获准后，我会使用工具（如修改代码、运行命令、安装依赖）来完成任务。
    *   Once approved, I'll use tools (edit code, run commands, install dependencies) to complete the task.
4.  **验证 (Verification)**: 我会运行测试或手动验证我的更改是否正确。
    *   I'll run tests or manually verify that my changes are correct.
5.  **提交 (Submission)**: 任务完成后，我会提交代码并说明更改内容。
    *   After completion, I'll submit the code with a description of the changes.
